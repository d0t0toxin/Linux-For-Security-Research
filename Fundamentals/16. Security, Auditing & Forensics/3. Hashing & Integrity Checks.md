# SHA256SUM Command in Linux

## Overview

The `sha256sum` command in Linux is used to compute and check SHA-256 hash values for files. SHA-256 is a cryptographic hash function that produces a 256-bit (32-byte) hash. This command is commonly used to verify file integrity.

## Syntax

```bash
sha256sum [OPTION]... [FILE]...
```

If no FILE is specified or if FILE is `-`, `sha256sum` reads from standard input.

## Options

- `-b`, `--binary` : read files in binary mode (default on Linux)
    
- `-t`, `--text` : read files in text mode
    
- `-c`, `--check` : read SHA-256 sums from a file and check them
    
- `--ignore-missing` : do not fail if a listed file is missing during check
    
- `--quiet` : do not print OK for each successfully verified file
    
- `--status` : do not output anything, status code shows success or failure
    
- `--strict` : exit non-zero for improperly formatted checksum lines in check mode
    
- `--tag` : create BSD-style checksum lines
    
- `-h`, `--help` : display help information
    
- `--version` : display version information
    

## Examples

### Generate SHA-256 Hash of a File

```bash
sha256sum filename.txt
```

### Generate SHA-256 Hash for Multiple Files

```bash
sha256sum file1.txt file2.txt
```

### Save SHA-256 Hash to a File

```bash
sha256sum filename.txt > checksum.sha256
```

### Verify File Integrity Using a Checksum File

```bash
sha256sum -c checksum.sha256
```

### Verify Multiple Files and Ignore Missing Files

```bash
sha256sum -c checksum.sha256 --ignore-missing
```

### Quiet Verification

```bash
sha256sum -c checksum.sha256 --quiet
```

### Use Status Mode

```bash
sha256sum -c checksum.sha256 --status
```

### Generate BSD-style SHA-256 Hash

```bash
sha256sum --tag filename.txt
```

### Read Input from Standard Input

```bash
echo "Hello World" | sha256sum
```

## Exit Status

- 0 : Success, all checksums match
    
- 1 : Failure, at least one checksum did not match
    
- 2 : Incorrect usage or other errors
    

## Notes

- SHA-256 is more secure than SHA-1 or MD5.
    
- Use `sha256sum` to verify downloaded files from the internet to ensure authenticity.
    
- The checksum file used with `-c` should follow the format: `[hash] [filename]`.

# CKSUM Command in Linux

## Overview

The `cksum` command in Linux calculates a CRC (Cyclic Redundancy Check) checksum and byte count for files. It is commonly used to verify data integrity and detect accidental changes in files.

## Syntax

```bash
cksum [OPTION]... [FILE]...
```

If no FILE is specified or if FILE is `-`, `cksum` reads from standard input.

## Options

- `-h`, `--help` : display help information and exit
    
- `--version` : output version information and exit
    

## Output Format

`cksum` produces three fields:

1.  CRC checksum
    
2.  Number of bytes in the file
    
3.  Filename
    

Example:

```
1234567890 1024 filename.txt
```

## Examples

### Calculate Checksum of a File

```bash
cksum filename.txt
```

### Calculate Checksum for Multiple Files

```bash
cksum file1.txt file2.txt
```

### Read Input from Standard Input

```bash
echo "Hello World" | cksum
```

### Using cksum with a File List

```bash
for file in *.txt; do cksum "$file"; done
```

## Notes

- `cksum` uses the POSIX standard CRC algorithm.
    
- It is primarily for detecting accidental changes, not for cryptographic security.
    
- The CRC value changes if the file content changes in any way, making it useful for integrity verification.
    
- Unlike `md5sum` or `sha256sum`, `cksum` does not produce cryptographic hashes and should not be used for security purposes.

# MD5DEEP Command in Linux

## Overview

The `md5deep` command in Linux is a utility for computing MD5, SHA-1, SHA-256, Tiger, and Whirlpool hashes of files. It supports recursive directory traversal and can compare hashes against a list, making it useful for file integrity verification across multiple files or directories.

## Syntax

```bash
md5deep [OPTION]... [FILE/DIRECTORY]...
```

If no FILE or DIRECTORY is specified, `md5deep` reads from standard input.

## Options

- `-r`, `--recursive` : recursively descend directories
    
- `-l`, `--follow` : follow symbolic links
    
- `-z`, `--zero` : separate output with null bytes instead of newline
    
- `-f FILE`, `--from FILE` : read list of paths from a file
    
- `-x FILE`, `--exclude FILE` : exclude paths matching a file or pattern
    
- `-X FILE`, `--exclude-from FILE` : exclude paths matching patterns in a file
    
- `-c FILE`, `--check FILE` : check hashes against a known list
    
- `-q`, `--quiet` : suppress non-essential output
    
- `-v`, `--verbose` : display verbose output
    
- `-h`, `--help` : display help information
    
- `--version` : display version information
    

## Examples

### Generate MD5 Hash of a File

```bash
md5deep filename.txt
```

### Generate MD5 Hash for Multiple Files

```bash
md5deep file1.txt file2.txt
```

### Recursive Hashing of a Directory

```bash
md5deep -r /path/to/directory
```

### Compare Files Against a Known Hash List

```bash
md5deep -c hashlist.txt
```

### Read List of Paths from a File

```bash
md5deep -f filelist.txt
```

### Exclude Specific Files or Patterns

```bash
md5deep -r -x exclude.txt /path/to/directory
```

### Quiet Mode

```bash
md5deep -q -r /path/to/directory
```

### Verbose Mode

```bash
md5deep -v filename.txt
```

### Use Standard Input

```bash
echo "Hello World" | md5deep
```

## Notes

- `md5deep` supports multiple hashing algorithms, not limited to MD5.
    
- It is particularly useful for bulk verification of file integrity across directories.
    
- The `-c` option can be used to compare current file hashes against a previously generated hash list for verification purposes.
    
- MD5 is not suitable for cryptographic security but is effective for file integrity checking.
