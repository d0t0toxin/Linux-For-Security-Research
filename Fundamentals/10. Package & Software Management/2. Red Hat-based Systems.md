# `dnf` Command in Linux

## Introduction

DNF stands for Dandified Yum. It is the modern package manager used by RPM based Linux distributions such as Fedora, Red Hat Enterprise Linux, Rocky Linux, AlmaLinux, and CentOS Stream. DNF is the successor to YUM and provides better performance, dependency resolution, and a cleaner command interface.

DNF is used to install, update, remove, search, and manage software packages from configured repositories.

## Basic Syntax

```
dnf [options] command [package...]
```

## Commonly Used DNF Commands

## Checking DNF Version

```
dnf --version
```

Displays the installed DNF version and system information.

## Listing Repositories

```
dnf repolist
```

Shows enabled repositories.

```
dnf repolist all
```

Shows both enabled and disabled repositories.

## Searching for Packages

```
dnf search nginx
```

Searches for packages related to the keyword nginx.

## Getting Package Information

```
dnf info nginx
```

Displays detailed information about a package including version, size, repository, and description.

## Installing Packages

```
sudo dnf install nginx
```

Installs the nginx package along with required dependencies.

Installing multiple packages:

```
sudo dnf install git curl wget
```

## Removing Packages

```
sudo dnf remove nginx
```

Removes the package but keeps unused dependencies if they are still required by other packages.

## Updating Packages

Update all installed packages:

```
sudo dnf update
```

Update a specific package:

```
sudo dnf update nginx
```

## Checking for Available Updates

```
dnf check-update
```

Lists packages that have updates available without installing them.

## Upgrading the System

```
sudo dnf upgrade
```

Performs a full system upgrade. Functionally similar to dnf update but recommended for modern usage.

## Reinstalling Packages

```
sudo dnf reinstall nginx
```

Reinstalls the specified package.

## Downgrading Packages

```
sudo dnf downgrade nginx
```

Downgrades a package to an earlier version if available in repositories.

## Listing Installed Packages

```
dnf list installed
```

Lists all installed packages.

## Listing Available Packages

```
dnf list available
```

Lists packages available in enabled repositories.

## Listing All Packages

```
dnf list all
```

Shows installed and available packages.

## Cleaning DNF Cache

```
sudo dnf clean all
```

Removes cached metadata and packages.

Clean only metadata:

```
sudo dnf clean metadata
```

## Managing Package Groups

List available groups:

```
dnf group list
```

Install a group:

```
sudo dnf group install "Development Tools"
```

Remove a group:

```
sudo dnf group remove "Development Tools"
```

## Enabling and Disabling Repositories

Enable a repository:

```
sudo dnf config-manager --set-enabled epel
```

Disable a repository:

```
sudo dnf config-manager --set-disabled epel
```

## Installing from a Local RPM File

```
sudo dnf install ./package-name.rpm
```

DNF automatically resolves dependencies from repositories.

## Handling Dependencies

Remove unused dependencies:

```
sudo dnf autoremove
```

## Viewing Transaction History

```
dnf history
```

Shows a list of all DNF transactions.

View details of a specific transaction:

```
dnf history info 12
```

Undo a transaction:

```
sudo dnf history undo 12
```

## Security Updates

```
sudo dnf update --security
```

Installs only security related updates.

## Excluding Packages from Updates

```
sudo dnf update --exclude=kernel*
```

Excludes kernel packages from being updated.

## Using DNF with Non Interactive Mode

```
sudo dnf install nginx -y
```

Automatically answers yes to all prompts.

## Verifying Installed Packages

```
dnf repoquery --installed
```

Displays installed packages using repository queries.

## Downloading Packages Without Installing

```
dnf download nginx
```

Downloads the RPM package without installing it.

## Troubleshooting and Debugging

Increase verbosity:

```
dnf -v install nginx
```

Enable debug output:

```
dnf -d 10 install nginx
```

## Configuration File

Main configuration file location:

```
/etc/dnf/dnf.conf
```

Repository files location:

```
/etc/yum.repos.d/
```

## Differences Between DNF and YUM

DNF is faster and more memory efficient  
DNF has better dependency resolution  
DNF supports automatic removal of unused dependencies  
DNF uses a modular design

## Best Practices

Keep system updated regularly  
Clean cache periodically  
Review transactions before confirming  
Avoid mixing repositories unnecessarily  
Use groups for development environments

## Conclusion

DNF is a powerful and reliable package manager for RPM based Linux systems. Mastering DNF allows efficient system maintenance, secure updates, and effective software management.

# YUM Command in Linux

## Introduction

YUM stands for Yellowdog Updater Modified. It is a package management tool used in RPM based Linux distributions such as CentOS, Red Hat Enterprise Linux, Oracle Linux, and older versions of Fedora. YUM simplifies software management by automatically resolving dependencies and interacting with configured repositories.

YUM was the default package manager before being replaced by DNF in newer systems. However, it is still widely used and important to understand, especially on legacy and enterprise systems.

## Basic Syntax

```
yum [options] command [package...]
```

## Checking YUM Version

```
yum --version
```

Displays YUM version and system details.

## Repository Management

### Listing Repositories

```
yum repolist
```

Shows enabled repositories.

```
yum repolist all
```

Shows enabled and disabled repositories.

### Enabling or Disabling Repositories

Enable a repository temporarily:

```
yum --enablerepo=epel install nginx
```

Disable a repository temporarily:

```
yum --disablerepo=epel update
```

## Searching for Packages

```
yum search httpd
```

Searches repositories for matching packages.

## Viewing Package Information

```
yum info httpd
```

Displays package details such as version, size, repository, and description.

## Installing Packages

Install a single package:

```
sudo yum install httpd
```

Install multiple packages:

```
sudo yum install git curl wget
```

## Removing Packages

```
sudo yum remove httpd
```

Removes the package and unused dependencies if not required elsewhere.

## Updating Packages

Update all packages:

```
sudo yum update
```

Update a specific package:

```
sudo yum update httpd
```

## Checking Available Updates

```
yum check-update
```

Displays packages with available updates.

## Upgrading the System

```
sudo yum upgrade
```

Performs a system wide package upgrade.

## Reinstalling Packages

```
sudo yum reinstall httpd
```

Reinstalls an installed package.

## Downgrading Packages

```
sudo yum downgrade httpd
```

Downgrades a package to an older available version.

## Listing Packages

List installed packages:

```
yum list installed
```

List available packages:

```
yum list available
```

List all packages:

```
yum list all
```

## Cleaning YUM Cache

```
sudo yum clean all
```

Removes cached packages and metadata.

Clean only metadata:

```
sudo yum clean metadata
```

## Group Management

List groups:

```
yum group list
```

Install a group:

```
sudo yum groupinstall "Development Tools"
```

Remove a group:

```
sudo yum groupremove "Development Tools"
```

## Installing Local RPM Packages

```
sudo yum localinstall package-name.rpm
```

Automatically resolves dependencies from repositories.

## Dependency Management

Remove orphaned dependencies:

```
sudo yum autoremove
```

## Transaction History

View transaction history:

```
yum history
```

View details of a transaction:

```
yum history info 5
```

Undo a transaction:

```
sudo yum history undo 5
```

Redo a transaction:

```
sudo yum history redo 5
```

## Security Updates

```
sudo yum update --security
```

Installs only security related updates.

## Excluding Packages from Updates

```
sudo yum update --exclude=kernel*
```

Prevents specified packages from updating.

## Non Interactive Mode

```
sudo yum install httpd -y
```

Automatically answers yes to prompts.

## Verifying Installed Packages

```
yum verify httpd
```

Checks integrity of installed package files.

## Downloading Packages Without Installing

```
yum install --downloadonly --downloaddir=/tmp httpd
```

Downloads RPM files without installation.

## Troubleshooting and Debugging

Increase verbosity:

```
yum -v install httpd
```

Enable debug mode:

```
yum -d 10 install httpd
```

## Configuration Files

Main configuration file:

```
/etc/yum.conf
```

Repository configuration directory:

```
/etc/yum.repos.d/
```

## Differences Between YUM and DNF

YUM is written primarily in Python  
DNF provides better dependency resolution  
DNF is faster and more memory efficient  
YUM is still used in legacy and long term support systems

## Best Practices

Keep repositories minimal and trusted  
Regularly update the system  
Clean cache periodically  
Review transaction history before undoing changes  
Avoid mixing incompatible repositories

## Conclusion

YUM remains a foundational tool for managing software on RPM based Linux systems. Understanding YUM is essential for maintaining older servers and enterprise environments where stability and long term support are critical.

# MicroDNF Command in Linux

## Introduction

MicroDNF is a lightweight version of the DNF package manager. It is designed for minimal container environments such as Docker and Podman, where reducing image size is critical. MicroDNF provides basic package management capabilities such as installing, updating, removing, and querying packages but with fewer features compared to full DNF.

MicroDNF is commonly used in Red Hat Universal Base Images and Fedora minimal images.

## Basic Syntax

```
microdnf [options] command [package...]
```

## Checking MicroDNF Version

```
microdnf --version
```

Displays MicroDNF version and system information.

## Installing Packages

Install a single package:

```
sudo microdnf install nginx
```

Install multiple packages:

```
sudo microdnf install git curl wget
```

MicroDNF automatically resolves dependencies from available repositories.

## Removing Packages

```
sudo microdnf remove nginx
```

Removes the specified package.

## Updating Packages

Update all installed packages:

```
sudo microdnf update
```

Update a specific package:

```
sudo microdnf update nginx
```

## Listing Installed Packages

```
microdnf list installed
```

Displays all installed packages.

## Listing Available Packages

```
microdnf list available
```

Displays packages available in enabled repositories.

## Searching for Packages

```
microdnf search nginx
```

Searches for packages matching the specified keyword.

## Getting Package Information

```
microdnf info nginx
```

Displays details about the package such as version, repository, and description.

## Cleaning Cache

```
sudo microdnf clean all
```

Removes cached metadata and package files.

## Installing from Local RPM

```
sudo microdnf install ./package-name.rpm
```

Installs a local RPM package and resolves dependencies from repositories.

## Non Interactive Mode

```
sudo microdnf install nginx -y
```

Automatically confirms all prompts without user interaction.

## Updating Repositories

```
sudo microdnf repolist
```

Lists available repositories.

```
sudo microdnf update -y
```

Updates packages from all enabled repositories.

## Downloading Packages Without Installing

```
microdnf download nginx
```

Downloads the package without installing it.

## Removing Unused Dependencies

```
sudo microdnf autoremove
```

Removes packages that were installed as dependencies but are no longer required.

## Differences Between MicroDNF and DNF

MicroDNF is minimal and lightweight  
MicroDNF is designed for containers  
MicroDNF has fewer features and plugins compared to DNF  
DNF provides full system management, MicroDNF is suitable for minimal environments

## Best Practices

Use MicroDNF in containerized environments  
Keep containers updated with security patches  
Remove unnecessary packages to keep images small  
Use official repositories to ensure stability and security

## Conclusion

MicroDNF is a simplified, efficient package manager for minimal Linux environments, particularly in containers. It provides essential package management functionalities with a small footprint, making it ideal for containerized applications and lightweight images.

# RPM Command in Linux

## Introduction

RPM stands for Red Hat Package Manager. It is a low-level package management tool used in RPM based Linux distributions such as Red Hat Enterprise Linux, CentOS, Fedora, Rocky Linux, and AlmaLinux. RPM is used to install, update, remove, verify, and query individual RPM packages.

Unlike YUM or DNF, RPM does not automatically resolve dependencies. It operates on individual package files.

## Basic Syntax

```
rpm [options] package-file
```

or for queries:

```
rpm [query options] [package-name]
```

## Checking RPM Version

```
rpm --version
```

Displays the installed RPM version.

## Installing Packages

Install an RPM package:

```
sudo rpm -i package-name.rpm
```

Force install (overwrite existing files):

```
sudo rpm -i --force package-name.rpm
```

Upgrade an existing package:

```
sudo rpm -U package-name.rpm
```

Freshen (upgrade only if package exists):

```
sudo rpm -F package-name.rpm
```

## Removing Packages

```
sudo rpm -e package-name
```

Removes the package from the system.

Force removal ignoring dependencies:

```
sudo rpm -e --nodeps package-name
```

## Querying Packages

List all installed packages:

```
rpm -qa
```

Get detailed information about a specific package:

```
rpm -qi package-name
```

List files installed by a package:

```
rpm -ql package-name
```

Check which package owns a file:

```
rpm -qf /path/to/file
```

List documentation files of a package:

```
rpm -qd package-name
```

List scripts executed during package installation or removal:

```
rpm -q --scripts package-name
```

Query packages by a specific pattern:

```
rpm -qa 'httpd*'
```

## Verifying Packages

Verify installed packages for integrity:

```
rpm -V package-name
```

This checks for file size, permissions, MD5 checksum, and other attributes.

Verify all packages:

```
rpm -Va
```

## Importing and Managing GPG Keys

Import a GPG key for verifying package signatures:

```
sudo rpm --import /path/to/RPM-GPG-KEY
```

List all imported keys:

```
rpm -qa gpg-pubkey*
```

Verify a package signature:

```
rpm --checksig package-name.rpm
```

## Building RPM Packages

Initialize build environment:

```
rpmdev-setuptree
```

Build an RPM from a spec file:

```
rpmbuild -ba package.spec
```

## Querying Package Database

List all packages with a summary:

```
rpm -qa --qf '%{NAME}-%{VERSION}-%{RELEASE}.%{ARCH}
'
```

## Checking Dependencies

Check dependencies of an RPM file:

```
rpm -qpR package-name.rpm
```

Check which packages require a specific package:

```
rpm -q --whatrequires package-name
```

Check what capabilities a package provides:

```
rpm -q --provides package-name
```

Check what capabilities a package requires:

```
rpm -q --requires package-name
```

## Verifying Database and Repair

Rebuild RPM database:

```
sudo rpm --rebuilddb
```

## Troubleshooting

Verbose installation:

```
sudo rpm -ivh package-name.rpm
```

This shows hash marks indicating progress.

Force dependencies ignored:

```
sudo rpm -i --nodeps package-name.rpm
```

## Configuration Files

RPM configuration file location:

```
/etc/rpm/macros
/etc/rpmrc
```

RPM database location:

```
/var/lib/rpm/
```

## Best Practices

Always verify packages with GPG keys  
Prefer using DNF or YUM for dependency resolution  
Keep RPM database healthy and rebuild if corrupted  
Check package integrity using rpm -V

## Conclusion

RPM is a powerful low-level package management tool for handling individual RPM files. Mastering RPM commands is essential for troubleshooting, managing packages without repository tools, and building custom RPM packages.
